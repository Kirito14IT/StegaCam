# 我的需求：设计一个app,首次打开软件要设置登录账号和密码，以及要选用的模型(现在暂时只有stega)，随机生成一个7位字母和数字组成的用户唯一id，这些先暂存在客户端，后期再用数据库存起来，现在不用数据库。

## 首页是<拍照>功能，客户端拍照后把图片和用户id(密文)传到后端编码，后端编码完后，保存图片为png要求无损传输回客户端，自动保存到相册,保证连拍的流畅性。首页右上角有小点表示是否连接服务器，拍照屏幕左上角是处理队列消息，拍照屏幕右上角是翻转相机的按钮。

## 手机屏幕中间是<选择>按钮，这个页面可以从相册选择图片进行解码，选择多张后放入列表等待点击开始解码，解码后显示密文,这个解码的页面逻辑可以参考图片Bigjpg的页面处理逻辑。

## 状态栏右下角是<设置>按钮，在这里显示用户名和密码以及id，还有一个下拉列表选择使用的处理模型(现在暂时只有stega)还有一个可以手动连接服务器的输入框(默认是服务器地址https:\\47.101.142.85:6100)，还有一些其他信息。随意发挥。
